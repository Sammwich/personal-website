Sum-up service using sockets

This application was created purely to practice udp and tcp sockets and multithreading. The application itself is quite senseless as it only adds up random integers, but the experience was good! Coding is an important skill for a data scientist and this helped me develop in object oriented programming. You can find the source code in my Github page.

The service consist of an application which connects to a server app using both UDP and DCP. After connecting, the application creates three threads which sum up integers sent by the server. The server application may ask for the sum of all the threads to which the application will answer. After the server stops sending integers the application does a controlled shut-down.

  • Coding language: Java
  • Date: November 2015
  • Type: Network programming

K-nearest neighbors algorithm

This is my first dip to machine learning. In order to understand the algorithm better I coded my own simple K-nearest neighbors algorithm. Tested with the Wisconsins breast cancer dataset You can find the code in my Github page.

KNN algorithm can be used for classification or regression. My alorithm example is used on classification. Wikipedia gives a quite nice and concise explanation about the algorithm. Put simply the algorithm predicts a class (in my data the tumor is either malignant or benign) for an instance (a tumor) based on it's neighbours(tumors that resemble it the most). Based on the neighbours class they 'vote' about which classification is given to the tumor at hand. For example: if we choose 5 nearest neighbours and 2 of them are benign and three are malignant, the tumor at hand is predicted to be malignant.

This was an interesting project to make and I look to dabbling more with machine learning algorithms. They're a good addition to a data scientist toolset!

  • Coding language: Python
  • Date: January 2016
  • Type: Machine Learning
